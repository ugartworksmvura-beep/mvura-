Accessibility

Last updated: 28 February 2026

Our Commitment

mvura is committed to ensuring digital accessibility for people with disabilities. We are continually improving the user experience for everyone and applying the relevant accessibility standards. Our goal is to make mvura.games accessible to the widest possible audience, regardless of technology or ability.

We believe that everyone should be able to enjoy games and connect with creators, and we strive to provide an inclusive environment for all our users.

Conformance Status

The Web Content Accessibility Guidelines (WCAG) define requirements for designers and developers to improve accessibility for people with disabilities. It defines three levels of conformance: Level A, Level AA, and Level AAA.

mvura aims to conform to WCAG 2.1 Level AA. These guidelines are internationally recognised and form the basis of accessibility regulations in many countries, including Malawi's emerging digital standards. We regularly test our website with automated tools and manual audits to identify and fix accessibility issues.

WCAG 2.1 AA We are actively working to meet these standards.

Accessibility Features

We have implemented the following features to improve accessibility:

  • Semantic HTML: We use proper heading structures, landmark regions, and semantic elements to ensure screen readers can navigate the site effectively.
  • Keyboard Navigation: All interactive elements (buttons, links, forms) are focusable and operable via keyboard. You can navigate using the Tab key, and activate elements with Enter or Space.
  • Skip Links: A "Skip to main content" link is available (visually hidden until focused) to allow keyboard users to bypass repetitive navigation.
  • Colour Contrast: We maintain sufficient colour contrast between text and background, meeting WCAG AA requirements. Our light and dark themes are both tested for contrast.
  • Text Resizing: The site uses relative units (rem) so that text can be resized up to 200% without loss of content or functionality. You can use browser zoom or text‑only zoom.
  • ARIA Attributes: We use ARIA landmarks and labels where necessary to enhance screen reader comprehension (e.g., for modal dialogs, dropdown menus).
  • Focus Indicators: All interactive elements have visible focus indicators (usually a coloured outline) to help keyboard users track their position.
  • Alternative Text: Images that convey information have descriptive alt text. Decorative images are marked with empty alt attributes.
  • Forms and Errors: Form inputs have associated labels, and validation errors are described clearly to assist users of assistive technology.
  • Dark Theme: A dark theme is available for users who prefer reduced brightness or have light sensitivity. The theme toggle is persistent across pages.

Limitations and Known Issues

Despite our best efforts, some parts of the website may not be fully accessible. We are aware of the following issues and are working to resolve them:

  • Third‑Party Content: Some games and content uploaded by developers may not be fully accessible. We encourage developers to follow accessibility best practices and provide guidance in our developer documentation.
  • Older PDFs: Some older documents (e.g., archived reports) may not be fully accessible. Please contact us if you need an alternative format.
  • Video Content: Some videos may lack captions or audio descriptions. We are working with content creators to improve this.

If you encounter any accessibility barriers, please report them using the contact information below.

Technical Specifications

mvura relies on the following technologies to work with the particular combination of web browser and any assistive technologies or plugins installed on your device:

  • HTML5
  • CSS3 (including Flexbox and Grid for layout)
  • JavaScript (enhancements, not required for basic navigation)
  • WAI‑ARIA for dynamic content

These technologies are relied upon for conformance with the accessibility standards used.

Assessment Methods

mvura assesses its accessibility through:

  • Automated testing: We use tools like axe, Lighthouse, and WAVE to scan our pages for common issues.
  • Manual testing: We perform keyboard‑only navigation and screen reader testing (using NVDA and VoiceOver).
  • User feedback: We listen to reports from users with disabilities and prioritise fixes.

Accessibility for Developers

If you are a developer publishing games on mvura, we encourage you to make your games accessible. Consider:

  • Providing subtitles/captions for any video or audio content.
  • Ensuring colour contrast meets WCAG standards.
  • Making your game fully playable with keyboard or alternative input methods.
  • Including options for text size adjustment and high‑contrast modes.
  • Testing with screen readers if your game includes text.

For more detailed guidance, please refer to our developer documentation or contact us at developers@mvura.games.

Ongoing Efforts

Accessibility is an ongoing process. We regularly review our website and make improvements. Some of the initiatives we are currently working on include:

  • Adding captions to all promotional videos.
  • Improving keyboard navigation for the game carousels.
  • Providing more descriptive labels for complex interactive components.
  • Creating an accessibility statement in local languages (Chichewa, etc.).

We are committed to making mvura a more inclusive platform for everyone.
